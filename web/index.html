deppy

deppy is an attempt to make software generation and verification feel like one pipeline, not two disconnected worlds. It treats intent, structure, semantics, proof, and code as layers of one typed object, then uses sheaf-theoretic local-to-global reasoning to find where they fail to glue.

Get started Read the math See the trading example

Real adoption

Run deppy on code you already have, with no annotation ceremony.

deppy .
deppy check src/ --format sarif

Spectacular generation

Use deppy as a prompt-driven builder that ideates, orchestrates, generates, and verifies.

deppy --ideation --orchestration --lean --strict --verbose "build a verified trading system that discovers a novel market idea and proves the risk-critical core"

Math-first verification

Intent/code gaps are modeled as cocycles and measured by cohomology.

\[H^1(U, \mathrm{HybridTy}) = 0 \Longleftrightarrow \text{the local layers glue into one coherent global section}.\]

What deppy is trying to do

  • Make existing Python code checkable with a zero-change entry point.
  • Lift natural-language specifications into typed, auditable generation plans.
  • Treat LLMs as controlled oracles rather than untyped magic boxes.
  • Compile structural claims to Z3 and Lean instead of leaving them as comments.
  • Use the same local-to-global mathematics for bug finding, equivalence, planning, and ideation.
