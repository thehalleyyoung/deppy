Hybrid dependent types

The product-site view of intent, structure, semantics, proof, and code.

On this page

A hybrid type is not just a syntactic annotation. It is a multi-layer typed object whose inhabitants must make sense across intent, structure, semantics, proof, and implementation.

\[\mathrm{HybridTy} : (\mathrm{Site}(P) \times \mathrm{Layer})^{op} \to \mathrm{RefinementLattice} \times \mathrm{TrustLattice}.\]

The layer set

  • INTENT
  • STRUCTURAL
  • SEMANTIC
  • PROOF
  • CODE

Dependent-type reading

  • Pi-types encode APIs and dependent interfaces.
  • Sigma-types encode module decompositions and project assembly.
  • Refinement types encode specifications and invariants.
  • Identity types encode compatibility on overlaps and integration proofs.
\[\Sigma\text{-types} \leadsto \text{module decomposition}, \qquad \Pi\text{-types} \leadsto \text{API contracts}.\]

Trust as part of the type

The same local section can be present with different trust levels: assumed, LLM-judged, property-checked, Z3-proven, Lean-verified, or human-verified.
