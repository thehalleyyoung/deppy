Examples

Verified trading example

The flagship prompt: a verified trading system with ideation and orchestration enabled.

On this page

This is the flagship demo because it exercises prompt resolution, ideation, typed orchestration, generation, verification, and project assembly.

deppy --ideation --orchestration --lean --strict --verbose "build a verified trading system that discovers a novel market idea and proves the risk-critical core"

What deppy does

  • Resolves the prompt into an intent object.
  • Runs ideation-as-typing over type theory, algebraic geometry, formal verification, and machine learning.
  • Selects a novel idea candidate for the trading architecture.
  • Builds a TT/AG orchestration cover over module sites.
  • Generates and verifies modules in cover order.
  • Checks cross-module gluing and reports H^1 on overlaps.

The typed module cover

market_data

Input site: normalize feeds and local observations.

signal_geometry

Ideation site: turn local signals into a gluable global section.

strategy_kernel

The Pi/Sigma core of the trading thesis.

risk_engine

Refinement site for exposure and drawdown invariants.

execution

Only risk-approved intents descend to executable orders.

portfolio

Dependent-record state over fills, cash, and positions.

compliance

Proof boundary and audit witness layer.

main

Global section that glues the sites into one runnable system.

The orchestration obligations

\[\mathrm{restrict}(s_i, U_i \cap U_j) = \mathrm{restrict}(s_j, U_i \cap U_j) \quad \text{for every adjacent module pair}.\]

The trading example is not just a list of modules; it is a cover with overlap conditions. The orchestration flag insists that integration is part of the type, not a postscript.

What makes it spectacular?

It is not merely generate eight files. It is discover a new idea, organize it as a typed local-to-global cover, generate the local sections, and audit the gluing data.
