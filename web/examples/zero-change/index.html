Zero-change verification

Use deppy on existing repositories without rewriting them into a new language first.

On this page

This is the adoption story: run deppy on code that already exists, without requiring a migration to a special proof language first.

deppy .
deppy check src/ --format sarif
deppy check src/ --incremental

Where the specifications come from

  • Type annotations and signatures - structural layer.
  • Docstrings and comments - intent layer.
  • Assertions and conditionals - refinement seeds.
  • Tests - semantic and structural evidence.
  • Names and code shape - heuristic intent hints.

The mathematical reading
