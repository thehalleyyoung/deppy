Architecture

Architecture

How deppy's classic pipeline, hybrid layers, bridges, and prompt-driven runtime fit together.

On this page

deppy is a stack of interacting subsystems: the original deppy analysis core, the hybrid dependent type extensions, the discharge and Lean pipeline, the type-expansion machinery, the ideation engine, and the prompt-driven runtime.

\[\text{prompt/spec} \to \text{intent} \to \text{typed cover} \to \text{local generation} \to \text{verification} \to \text{glued project}.\]

Subsystems

Classic core

Presheaves, sites, refinement typing, pipeline stages, renderers.

Hybrid core

Trust, provenance, contracts, hybrid type checking, product-site reasoning.

Discharge

Z3, oracle, Lean, and residual obligation routing.

Type expansion

Expand NL specs into module decompositions and generation plans.

Ideation

Cross-domain analogy checking and self-improvement loops.

Agent runtime

Intent resolution, generation, CEGAR, and project assembly.

Layer stack

  • INTENT - natural language and user goals
  • STRUCTURAL - signatures, schemas, refinements, decidable constraints
  • SEMANTIC - tests, meaning, behavior, oracle-mediated claims
  • PROOF - Lean obligations and proof artifacts
  • CODE - the executable implementation
