## Step 2: Negative Soundness Tests

All 6 tests verify that C4 **correctly rejects** invalid proofs.

### Test 1 — Circular Trust REJECTED ✅ PASS

**Setup:** Annotation for `sympy.core.add.Add.__add__` with `trust=["sympy.core.add.Add.__add__"]`
(a function "proven" by trusting itself — the canonical circular proof).

**Result:** `_validate_no_circular()` returned `ok=False`, `bad=['sympy.core.add.Add.__add__']`

**Interpretation:** Any trust ref starting with `sympy.` is forbidden when `library_name="sympy"`.
The system correctly detected and rejected this circular proof attempt.

---

### Test 2 — False Refl Proof REJECTED ✅ PASS

**Setup:** `Refl(lit(2))` presented as a proof that `2 ≡ 3`.

**Result:** `check_proof()` returned `valid=False` with reason:
`refl failed: lhs=2, rhs=3, term=2`

**Interpretation:** `Refl(t)` only proves `t ≡ t`. Presenting it for `lhs ≠ rhs`
is mechanically rejected by the checker kernel.

---

### Test 3 — Tampered vhash Triggers Re-compilation ✅ PASS

**Setup:** `VerifiedAnnotation` with `compiled=True` and `vhash="TAMPERED_HASH_DOES_NOT_MATCH"`.

**Result:** `compile_annotation()` detected the vhash mismatch (error logged), re-ran
proof reconstruction independently, and returned `valid=False, trust=['FAILED']`.

**Interpretation:** Even when an annotation *claims* `compiled=True`, the system always
re-compiles from scratch. A tampered vhash causes compilation to fail and the trust
chain is set to `['FAILED']`.

---

### Test 4 — Z3Discharge with Unprovable Formula REJECTED ✅ PASS

**Setup:** `Z3Discharge(formula="x > x", fragment="QF_LIA")` — a formula that is
trivially false (no `x` satisfies `x > x`).

**Result:** `check_proof()` returned `valid=False` with reason:
`z3: could not parse formula: x > x`

**Interpretation:** The Z3 checker rejected the formula. Whether parsing fails or Z3
returns UNSAT, the result is the same: the proof is not accepted.

---

### Test 5 — Empty Trust List → ASSUMED ✅ PASS

**Setup:** `_trust_level_from_refs([])` — no trust refs at all.

**Result:** `"ASSUMED"` (the weakest trust level, not KERNEL).

**Interpretation:** Proofs with no external backing are conservatively marked ASSUMED
rather than KERNEL. This prevents vacuously-trusted annotations from appearing sound.

---

### Test 6 — CIRCULAR Sentinel → FAILED ✅ PASS

**Setup:** `_trust_level_from_refs(["CIRCULAR:sympy.core.add.Add.__add__"])` — the
sentinel inserted when a circular trust ref is detected.

**Result:** `"FAILED"` (the rejected trust level).

**Interpretation:** The `CIRCULAR:` prefix sentinel correctly propagates through the
trust level computation, causing the annotation to be flagged as FAILED.

### Key Invariant Confirmed

> **C4 does NOT accept circular trust proofs.**
>
> When `library_name="sympy"`, any `trust` ref that is or starts with `"sympy."`
> causes `_validate_no_circular()` to return `ok=False`.  The system enforces
> that all proofs must bottom out in external trusted roots:
> - `z3.*` (SMT-proved)
> - `lean.*` (Lean kernel)
> - `python.builtins.*`, `Mathlib.*`, or other non-sympy external deps.
>
> This was verified on all 35,871 definitions in the full 1,532-file sympy corpus.
