Python with Dependent Types

Deppy brings formal verification to the world's most popular language. Using dependent type theory, Z3 SMT solving, and sidecar proofs, you can verify existing Python code — without rewriting a single line.

Start Reading →

About This Book

This book is a tutorial and reference for Deppy, a proof-oriented verification system for Python. It is modeled after the F* Tutorial Book, but with a crucial difference: where F* is a language you write new code in, Deppy verifies the Python code you already have.

We cover everything from basic refinement types and Z3 integration to advanced topics like homotopy-theoretic path types, Čech decomposition for modular verification, separation logic for heap reasoning, and GIL-aware concurrency proofs.

Deppy does not require you to modify your source code. Specifications and proofs live in sidecar .deppy files alongside your Python modules.

A Quick Example

Here is a simple Python function and its Deppy verification. The source code remains untouched — the specification lives in a sidecar file.

my_math.py

# Ordinary Python — no Deppy imports
def abs_val(x: int) -> int:
    if x >= 0:
        return x
    return -x

my_math.deppy

# Sidecar proof — sits beside my_math.py
from deppy import verify, refined

@verify("my_math.abs_val")
def abs_val_spec(x: int) -> refined(int, lambda r: r >= 0):
    # Postcondition: result ≥ 0
    # Proof: case split on x ≥ 0
    z3.auto()  # discharged by Z3

Running deppy check my_math.py verifies the specification without executing any code:

$ deppy check my_math.py
✓ my_math.abs_val  — Z3_PROVEN (0.02s)
  postcondition: result ≥ 0
  strategy: case-split → Z3 auto
All 1 obligations verified.

Why Deppy?

Existing proof-oriented systems require you to learn a new language and rewrite your code. Deppy meets Python developers where they are.

A Taste of Verification

Deppy supports several styles of specification. Here's a richer example using the @guarantee decorator for inline annotation and @refined for refinement types: 

from deppy import guarantee, refined, assume

Nat = refined(int, lambda n: n >= 0)

@guarantee("result == n * (n + 1) // 2")
def triangle(n: Nat) -> Nat:
    """Sum of integers from 0 to n."""
    return n * (n + 1) // 2
$ deppy check triangle.py
✓ triangle  — Z3_PROVEN (0.01s)
  guarantee: result == n * (n + 1) // 2
  refinement: result ≥ 0
All 2 obligations verified.

Verification is fast. Z3 handles arithmetic obligations automatically, and Deppy decomposes complex proofs into manageable pieces using Čech decomposition from algebraic topology.
