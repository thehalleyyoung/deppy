\begin{abstract}
\noindent
This monograph develops a verification methodology for \emph{imported}
Python libraries.  The user names a library --- e.g. \texttt{import
sympy.geometry} --- and supplies a sidecar specification file
(\texttt{.deppy}) carrying typed property annotations and proof scripts.
The \Deppy{} pipeline introspects the live module, parses the sidecar,
elaborates each verification block into a \emph{proof term} in
\Deppy's kernel, discharges arithmetic obligations to \Zthree, runs
counterexample search on the negation, and emits a self-contained
\Lean{}~4 certificate that round-trips through Lean's elaborator.

The proof scripts are written in \PSDL{}, the \emph{Python-Semantic
Deppy Language}: a Python-syntax tactic language whose vocabulary
includes both \CIC{}-flavoured primitives (induction, transport,
function extensionality, path composition) \emph{and} primitives
intrinsic to reasoning about cubical-duck-typed effectful Python
(\texttt{isinstance} fibrations, exception fibres under the
$\Diamond/\Box$ modalities, monkey-patch paths, Kan fillers,
$\Sigma$-projection through \texttt{\_\_getattr\_\_}).  Each \PSDL{}
statement compiles in lock-step to a \Deppy{} \texttt{ProofTerm} that
the kernel checks and a \Lean{} tactic that Lean's kernel checks.

We work out the design in detail, present the formal model, document
every component of the pipeline against the source tree, and walk
through the worked example of verifying properties of
\texttt{sympy.geometry.point.Point.distance} end-to-end.  We are
explicit about the trust surface: foundations admit, opaque
\texttt{axiom}s admit, \texttt{Structural} leaves admit; the
certificate is honest about which subgoals it has actually
discharged and which it has merely passed through.
\end{abstract}
