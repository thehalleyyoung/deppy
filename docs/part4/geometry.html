Verifying sympy.geometry: Euclidean Theorems

This chapter walks through a complete, runnable sidecar verification of SymPy's computational geometry module. We prove 24 classical theorems of Euclidean geometry — Heron's formula, the Euler line, the nine-point circle, and more — hold in SymPy's implementation, without modifying a single line of SymPy's source code.

One theorem is deliberately false. The system catches it, proving that the verification pipeline is not rubber-stamping everything.

Try it yourself. The full working example lives at verify_sympy_geometry/ in the deppy repo. Run it with: 
PYTHONPATH=. python3 verify_sympy_geometry/verify_triangle.py
Result: 23/23 kernel proofs, 182/182 runtime checks, 1 expected failure — in ~26 000 ms.

What We're Proving

The verification covers seven categories of Euclidean geometry, each backed by axioms in a .deppy sidecar file and validated computationally on a diverse family of exact-coordinate triangles:

Category # Theorems
A. Distance4Non-negativity, symmetry, identity, triangle inequality
B. Area4Cross-product formula, Heron's formula, medial ¼ theorem, orientation
C. Euler Line6Circumcenter equidistance, R = abc/4A, centroid averaging, collinearity, OG:GH = 1:2, incenter
D. Nine-Point Circle4Center = midpoint(O, H), radius = R/2, passes through midpoints, altitude feet
E. Circle2Area = πr², circumference = 2πr
F. Similarity3Angle-preserving similarity, side-preserving congruence, circumscription
G. Soundness1"Every triangle is equilateral" — correctly refuted

The .deppy Sidecar File

The sympy_geometry.deppy file declares axioms about SymPy without importing or modifying it. Here is the section for the Euler line: 

# sympy_geometry.deppy — Euler line axioms

spec sympy.geometry.Triangle.euler_line:
  guarantee: "circumcenter, centroid, and orthocenter are collinear"
  pure: true
  axiom euler_collinear: "is_collinear(circumcenter, centroid, orthocenter)"
  axiom euler_ratio: "dist(O, G) : dist(G, H) == 1 : 2"

And here is the deliberately false axiom — a soundness tripwire:

# This MUST fail computational validation
spec sympy.geometry.Triangle.is_equilateral:
  guarantee: "every triangle is equilateral"
  axiom all_equilateral: "Triangle(A,B,C).is_equilateral() == True for all A,B,C"

The full file declares 28 axioms across 14 specs, covering distance, area, triangle centres, the nine-point circle, circles, and similarity.

The Verification Script

The companion verify_triangle.py does three things for every theorem:

1
Install the axiom into deppy's proof kernel via library_trust.
2
Prove via the kernel — the axiom is accepted and recorded at AXIOM_TRUSTED level.
3
Validate computationally on 8 diverse triangle families using exact arithmetic (Rational, sqrt) — never floats.

Triangle Test Families

Every theorem is checked against these 8 non-degenerate, exact-coordinate triangles — chosen to cover right, isosceles, equilateral, scalene, rational, negative-coordinate, and obtuse cases: 

def _test_triangles() -> list[tuple[str, Triangle]]:
    return [
        ("3-4-5 right",
         Triangle(Point(0, 0), Point(4, 0), Point(0, 3))),
        ("isoceles 6-6-4",
         Triangle(Point(0, 0), Point(4, 0), Point(2, sqrt(32)))),
        ("equilateral side 2",
         Triangle(Point(0, 0), Point(2, 0), Point(1, sqrt(3)))),
        ("scalene (1,1)-(5,2)-(3,7)",
         Triangle(Point(1, 1), Point(5, 2), Point(3, 7))),
        ("large right 5-12-13",
         Triangle(Point(0, 0), Point(12, 0), Point(0, 5))),
        ("rational coords",
         Triangle(Point(Rational(1,3), Rational(2,5)),
                  Point(Rational(7,2), Rational(1,4)),
                  Point(Rational(5,6), Rational(11,3)))),
        ("negative coords",
         Triangle(Point(-3, -1), Point(4, -2), Point(1, 5))),
        ("wide obtuse",
         Triangle(Point(0, 0), Point(10, 0), Point(5, 1))),
    ]

Highlight: The Euler Line

The Euler line is one of the most beautiful results in triangle geometry: for any non-equilateral triangle, the circumcenter O, centroid G, and orthocenter H are collinear, and O, G, H divide that line in ratio 1:2.

Here's how we verify both properties:

# C4. Euler line: O, G, H collinear
r = (Proof("circumcenter, centroid, orthocenter are collinear")
     .using(kernel).given(T="object")
     .by_axiom("euler_collinear", "sympy.geometry").qed())

for name, tri in _test_triangles():
    if tri.is_equilateral():
        continue  # all centres coincide — ratio is 0/0
    cc = tri.circumcenter
    cg = tri.centroid
    oc = tri.orthocenter
    assert Point.is_collinear(cc, cg, oc)

# C5. Euler ratio OG:GH = 1:2
for name, tri in _test_triangles():
    if tri.is_equilateral():
        continue
    og = tri.circumcenter.distance(tri.centroid)
    gh = tri.centroid.distance(tri.orthocenter)
    assert simplify(og / gh) == Rational(1, 2)

This checks every non-equilateral triangle in our test family using SymPy's exact symbolic arithmetic — no floating-point approximation.

Highlight: The Nine-Point Circle

The nine-point circle passes through nine special points of a triangle: the three side midpoints, the three altitude feet, and the three midpoints of segments from vertices to the orthocenter. We verify four of its properties: 

# D1. NPC center = midpoint(circumcenter, orthocenter)
for name, tri in _test_triangles():
    npc = tri.nine_point_circle
    cc  = tri.circumcenter
    oc  = tri.orthocenter
    mid = Point((cc.x + oc.x) / 2, (cc.y + oc.y) / 2)
    assert npc.center == mid

# D2. NPC radius = circumradius / 2
for name, tri in _test_triangles():
    assert simplify(tri.nine_point_circle.radius
                    - tri.circumradius / 2) == 0

# D3. NPC passes through side midpoints
for name, tri in _test_triangles():
    npc = tri.nine_point_circle
    for side in tri.sides:
        mp = side.midpoint
        assert simplify(npc.center.distance(mp) - npc.radius) == 0

Highlight: Heron's Formula

Heron's formula computes a triangle's area from its side lengths alone:

|A| = √(s(s − a)(s − b)(s − c))   where   s = (a + b + c) / 2 

for name, tri in _test_triangles():
    sl = [s.length for s in tri.sides]
    s_semi = sum(sl) / 2
    heron = sqrt(s_semi * (s_semi - sl[0])
                         * (s_semi - sl[1])
                         * (s_semi - sl[2]))
    # Compare squares to avoid √ simplification issues
    assert simplify((Abs(tri.area) - heron)**2) == 0
Tip: When comparing expressions involving square roots, compare squares instead. SymPy's simplify sometimes can't reduce √a − √b to zero, but it can always reduce (√a − √b)² to zero when a = b.

Soundness Check: The False Axiom

A verification system that accepts everything is useless. To demonstrate that deppy's computational layer actually catches errors, we include a deliberately false axiom: 

# In the .deppy sidecar file:
spec sympy.geometry.Triangle.is_equilateral:
  guarantee: "every triangle is equilateral"
  axiom all_equilateral: "Triangle(A,B,C).is_equilateral() == True for all A,B,C"

The kernel accepts this as a trusted axiom — that's by design, since axioms are the starting point for reasoning. But the runtime validation immediately refutes it: 

# In verify_triangle.py:
counter_tri = Triangle(Point(0, 0), Point(5, 0), Point(1, 2))
assert not counter_tri.is_equilateral()  # ← refutes the axiom

Output:

  ⚠  Axiom 'all_equilateral' is trusted by the kernel —
     but computational validation REFUTES it:
    runtime: △(0,0)(5,0)(1,2) is equilateral? (expect False) ✓
  ✗ (expected) [RUNTIME_REFUTED     ] G1  'every triangle is equilateral' — REFUTED
Lesson: Axioms are powerful but dangerous — they let you assume anything. Always pair axiom-based proofs with computational validation. If the validation contradicts the axiom, the axiom is wrong. In a production workflow, you would either remove the axiom or investigate the source of the contradiction.

Design Decisions

Why Exact Arithmetic?

All coordinates use Rational or sqrt — never Python floats. This means that when SymPy says simplify(expr) == 0, it is an exact algebraic identity, not a floating-point approximation. This is what elevates our runtime checks from "testing" to "validation on the algebraic closure of ℚ."

Why Abs(area)?

SymPy's Triangle.area is signed — it depends on the orientation of the vertices. Reversing vertex order negates the area. Theorems like Heron's formula and R = abc/4A need the unsigned area, so we use Abs(tri.area).

Why Skip Equilateral for Euler Ratio?

In an equilateral triangle, the circumcenter, centroid, and orthocenter all coincide at the same point. The Euler ratio OG:GH becomes 0/0 — undefined. The collinearity claim is vacuously true (a single point is collinear with itself), but the ratio test must be skipped.
